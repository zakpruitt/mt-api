Get started

    API Endpoint

        https://mt-api.tech/api/v1

The MT (Monster Train) API provides a programmatic access to machine readable data from Shiny Shoe's Monster Train. For example, this API enables a developer to iterate through and access hundreds of cards, artifacts, enemies, etc within seconds.

MT API is organized and built with REST methodologies. This API has predictable resource-oriented URLs, returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

MT API and myself are not affilitated whatsoever with Monster Train or Shiny Shoe, making the MT API unofficial. All information distributed here is provided as-is and I (the author) cannot be responsible for your use of the information contained in or linked from mt-api.tech.

To clarify: to use this API, you do not need an API key. I strongly recommend you use Postman to perform requests (outside of any programming language).

MT API is hosted on a free Heroku dyno and is secured through Cloudfare; thus, when querying the API, you may need to wait for the dyno to spin up. Everything is cached, so once the dyno is ready to go, you should be all set!

To access the GitHub repository, click the link in the top right! Feel free to submit improvements. If you find any bugs or erroneous data, please email me at pruittzn@gmail.com.

GET Artifacts


# curl example for get all artifacts
curl -X GET "https://www.mt-api.tech/api/v1/artifacts" -H "accept: */*"


# curl example for a specific artifacts
curl -X GET "https://www.mt-api.tech/api/v1/artifacts/Mind%20Hoard" -H "accept: */*"

To get all artifacts, you need to make a GET call to the following url :
https://mt-api.tech/api/v1/artifacts

To get a specific artifact, you need to make a GET call to the following url :
https://mt-api.tech/api/v1/artifacts/{artifact name}

CARD MODEL (SUPER MODEL)

Field Type Description
cardLore String The in-game lore of the card.
cardName String The card's name. This field is also used to request a specific card (in other requests).
description String The card's description. A description is what effects a card has.
emberCost Integer The card's in-game ember cost.

imageURL String The card's official imageURL.
rarity Integer The card's in-game rarity value.
subtype String The card's subtype.
type String The card's type.
